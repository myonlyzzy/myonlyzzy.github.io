###ClientConn处理连接的过程

当Server.Run 最终会调用ClientConn的Run方法来处理连接到Server的连接。所以ClientConn中的方法基本包含了处理连接的逻辑。

  1. func (s *Server) onConn(c net.Conn)

Server的onConn方法是将连接到Server实例的连接转换成ClientConn。然后调用ClientConn的Run方法

  1. func (c *ClientConn) Run()

    Run方法在一个永真for循环中调用readPacket方法读取数据包再交给dispatch方法处理。

  2. func (c *ClientConn) dispatch(data []byte) error

dispatch方法负责通过解析数据包的第一个字节并根据第一个字节的内容在一个switch 中选择具体的处理函数

  1. func (c *ClientConn) handleQuery(sql string) (err error)

handleQuery方法负责处理cmd为COM_QUERY为3的的sql也是用的最多的sql命令。在handleQuery调用preHandleShard方法来预处理一下sql语句。

  1. func (c *ClientConn) preHandleShard(sql string) (bool, error)

    preHandleShard方法会先根据sql语句利用空格等分割生成一个string slice。再调用GetExecDB方法来构造一个ExecDB。再调用getBackendConn方法来从初始Server的时候建立的TCP连接中取的一个可用的连接。最后调用executeInNode方法来执行sql。根据放回的rs长度调用writeResultset或者WriteOK方法发送数据包到应用端。

  2. func (c *ClientConn) GetExecDB(tokens []string, sql string) (*ExecuteDB, error) 中会根据preHandleShard生成的slice swich不同的get函数 select 会到getSelectExecDB。
